## Описание

`mocking` описывает тестовые mock-объекты.

## Правила

- Mock-объекты размещай в зеркальной структуре относительно подменяемого слоя.
- Mock повторяет публичный контракт подменяемого клиента или сервиса.
- Если mock используется в нескольких тестах, храни его в `tests/mocking`, а не внутри одного тестового файла.
- Не добавляй в mock поведение, которого нет в публичном контракте реальной зависимости.

## Примеры

```python
class Redis:
    async def start(self) -> None:
        pass

    async def shutdown(self) -> None:
        pass

    async def get(self, _: str) -> dict[str, typing.Any]:
        return {}
```
