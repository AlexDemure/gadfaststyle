## Описание

`tools` описывает общие тестовые инструменты проекта.

## Правила

- В `tools` держи профилировщики, вспомогательные утилиты, контекстные менеджеры и отчетные модели для тестов.
- Инструменты должны быть переиспользуемыми и не зависеть от одной конкретной фичи.
- Если код нужен только одному тесту, не выноси его в `tests/tools/`.
- Общие инструменты нагрузки и диагностики подключай отсюда в `test_loads`, `test_integrations` и `test_units`.

## Примеры

```python
from tests.tools.profiler import reports
```
