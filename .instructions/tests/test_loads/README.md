## Описание

`test_loads` описывает нагрузочные тесты проекта.

## Правила

- В `test_loads` размещай сценарии нагрузки и профили производительности, а не функциональные проверки.
- Нагрузочные тесты не должны дублировать интеграционные и unit-тесты один в один.
- Конфигурацию профиля нагрузки и входные параметры держи рядом со сценарием теста.
- Если тест использует общие инструменты, выноси их в `tests/tools/`.

## Примеры

```python
from tests.tools.profiler import profilers
```
