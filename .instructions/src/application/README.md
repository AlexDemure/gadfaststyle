## Описание

`application` описывает прикладной слой.

## Правила

- `application` оркестрирует сценарий и не знает HTTP-деталей.
- Usecase вызывает доменные модели, adapters и security через уже принятые зависимости.
- Не вводи обязательный агрегирующий экспорт через `__init__.py`, если пакет этого уже не делает.
- Новую общую прикладную логику выноси в `utils/` только если она реально переиспользуется.

## Примеры

```python
from src.application.usecases.accounts.create import Usecase
```
