## Описание

`configuration` описывает конфигурацию приложения.

## Правила

- Конфигурацию оформляй отдельными settings-моделями и setup-функциями.
- Чтение переменных окружения и сборку конфигурации держи здесь, а не в `bootstrap` и не в `entrypoints`.
- Конфигурация не должна содержать бизнесовую логику и побочные эффекты рантайма.
- Доступ к конфигурации в других слоях делай через уже принятый setup или публичный экспорт пакета.

## Примеры

```python
from src.configuration import setup
```
