## Описание

`decorators` описывает cross-cutting декораторы проекта.

## Правила

- В `decorators` держи только переиспользуемые обертки, которые не зависят от конкретной бизнесовой сущности.
- Декоратор меняет техническое поведение вызова, но не содержит бизнесовую логику.
- Декораторы usecase-слоя должны работать через принятые зависимости и контракты проекта.
- Не дублируй один и тот же декоратор в нескольких слоях.

## Примеры

```python
from src.decorators.usecases.session import session
```
