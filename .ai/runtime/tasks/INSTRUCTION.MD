## Назначение

`runtime/tasks` хранит исполняемые артефакты агентного конвейера по каждой задаче.

## Структура

- Для каждой задачи создается папка `.ai/runtime/tasks/<YYYY-MM-DD>_<slug>/`.
- Внутри папки каждый этап пишет отдельный markdown-файл.
- Базовые имена этапов: `10-delivery-manager.md`, `20-project-architect.md`, `30-task-orchestrator.md`, `40-code-implementer.md`, `50-test-writer.md`, `60-code-reviewer.md`, `70-report-compiler.md`.
- При повторных циклах доработки используй следующий свободный номер внутри десятка: `41-code-implementer.md`, `51-test-writer.md`, `61-code-reviewer.md` и так далее.

## Правила

- Один запуск задачи должен писать артефакты только в одну runtime-папку.
- Если этап не выполнен, вместо пропуска создавай файл с блокером или явной причиной.
- Для циклических этапов не создавай больше 5 файлов на один тип этапа, включая первую итерацию.
- Отчетные файлы должны быть пригодны и для чтения человеком, и для последующей автоматической обработки.
