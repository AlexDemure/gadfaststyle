# Application Usecase: Create

## Когда использовать

Используй `create`, когда сценарий создает новую сущность, новую связь или новый аутентификационный артефакт.

## Что обычно делает create-usecase

- Собирает зависимости через `build(session)`.
- Проводит прикладные проверки перед записью.
- При необходимости использует `security` или format helpers.
- Создает доменную модель через `Model.init(...)` или близкий текущий паттерн.
- Передает создание в repository.
- Возвращает доменную модель, DTO или токены, если это соответствует текущему домену.

## Базовые правила

- Используй `@sessionmaker.write`.
- Для предварительных проверок добавляй `async def validate(...)`.
- `validate(...)` вызывает только нужные repository-проверки, чаще всего `exists()`, и поднимает `AlreadyExists` или `NotFound`.
- `validate(...)` ничего не возвращает и принимает только аргументы, нужные для проверки.
- Проверки на уникальность и конфликт состояния выполняй до `create(...)`.
- Если требуется шифрование, нормализация или вычисление производных полей, делай это в usecase только в пределах orchestration.
- Поднимай доменные исключения вроде `AlreadyExists`, если конфликт ожидаемый и бизнесовый.
