# Application Usecase: Get

## Когда использовать

Используй `get`, когда сценарий читает одну сущность или одно состояние по конкретному идентификатору, ключу или типу доступа.

## Что обычно делает get-usecase

- Собирает зависимости через `build(session)`.
- Принимает один способ идентификации или несколько близких вариантов в отдельных файлах внутри `get/`.
- Делает минимальную прикладную обработку входа, если это уже принято в домене.
- Вызывает repository для чтения одной записи.
- Проводит пост-проверки состояния, если они бизнесовые.

## Базовые правила

- Используй `@sessionmaker.read`.
- Разные способы чтения разделяй по разным файлам внутри `get/`, например `external.py`, `jwt.py`.
- Не смешивай `get` и `search`: `get` возвращает одну сущность, `search` возвращает коллекцию.
- Если после чтения требуется бизнесовая проверка, например `blocked`, делай ее в usecase.

## Эталон

Ориентируйся на `src/application/usecases/accounts/get/`.
