# Application Usecase: Delete

## Когда использовать

Используй `delete`, когда сценарий удаляет сущность или переводит ее в удаленное состояние.

## Что обычно делает delete-usecase

- Собирает зависимости через `build(session)`.
- Принимает идентификатор или минимальный набор входных параметров.
- Вызывает repository-операцию удаления без лишней прослойки.
- При необходимости делает предварительные проверки существования или прав доступа, если они уже приняты в домене.

## Базовые правила

- Используй `@sessionmaker.write`.
- Для предварительных проверок добавляй `async def validate(...)`, если перед удалением нужно проверить существование сущности или допустимость операции.
- `validate(...)` вызывает только нужные repository-проверки, чаще всего `exists()`, и поднимает `NotFound` или другие доменные ошибки.
- `validate(...)` ничего не возвращает и принимает только аргументы, нужные для проверки.
- Не раздувай delete-usecase логикой, если рядом удаление делается прямым вызовом repository.
- Если удаление мягкое, usecase все равно не должен знать о SQL-деталях реализации.

## Эталон

Ориентируйся на `src/application/usecases/accounts/delete.py`.
