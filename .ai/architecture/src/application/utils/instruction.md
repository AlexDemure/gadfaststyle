# Architecture: Application Utils

## Что входит в раздел

- `src/application/utils/<domain>.py`

## Назначение

`application/utils` содержит прикладные утилиты, которые часто переиспользуются между несколькими usecase одного домена или нескольких близких сценариев.

## Как организовывать

- Для каждого домена создавай отдельный файл:
  - `src/application/utils/account.py`
  - `src/application/utils/shelter.py`
  - `src/application/utils/city.py`
- Внутри файла размещай только те функции, которые действительно переиспользуются в usecase.
- Если логика нужна одному usecase и не выглядит как стабильное переиспользование, оставляй ее в самом usecase.

## Что здесь хранить

- Повторяющиеся прикладные преобразования.
- Общие шаги подготовки данных перед вызовом repository или после чтения модели.
- Переиспользуемые функции, которые снимают дублирование между usecase одного домена.

## Что здесь не хранить

- HTTP-логику.
- SQLAlchemy statement и прямую работу с таблицами.
- Случайные helper-функции без явного переиспользования.
- Логику, которая по смыслу должна жить в domain-модели или infrastructure.

## Правила написания

- Группируй утилиты по доменному файлу, а не по типу функции.
- Перед добавлением новой утилиты проверь существующий файл домена и соседние usecase.
- Не выноси код в `utils` преждевременно: сначала должно быть понятное переиспользование.
- Сигнатуры и нейминг должны оставаться прикладными и понятными по месту вызова в usecase.

## Эталон

Ориентируйся на `src/application/utils/account.py`.
