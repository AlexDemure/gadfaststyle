## Правила

- Если сценарий должен проверить существование сущности или допустимость удаления, используй `validate(...)`.
- Не раздувай `delete` дополнительной логикой, если сценарий сводится к прямому вызову repository.
- Если удаление мягкое, usecase не должен знать детали persistence-реализации.

## Допустимо

- Проверка существования через repository `exists(...)` или `one(...)`.
- Возврат `None`, если сценарий удаления не требует тела ответа.
