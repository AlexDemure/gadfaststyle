## Правила

- Knowledge, spec, engine и runtime разделяются по разным пакетам.
- Markdown-инструкции хранятся только в `knowledge/`.
- Machine-readable конфигурация хранится только в `spec/`.
- Исполняемый код агентного runtime хранится только в `engine/`.
- Артефакты запусков и служебное состояние задач хранятся только в `runtime/`.
- Engine должен опираться на контракт из `spec/`, а не на жестко прошитый в коде список knowledge-файлов.
- Knowledge-слой не должен зависеть от конкретной реализации engine.
