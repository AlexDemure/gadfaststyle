# AI Instructions Layout

- `.ai/rules/` - базовые правила для всех агентов.
- `.ai/architecture/` - проектные инструкции по слоям и общая карта архитектуры.
- `.ai/agents/` - роли агентов и их рабочие процессы.
- `.ai/reports/` - формат хранения артефактов и итоговых отчетов.

Текущий рекомендуемый конвейер:

1. `10-delivery-manager`
2. `20-project-architect`
3. `30-task-orchestrator`
4. `40-code-implementer`
5. `50-test-writer`
6. `60-code-reviewer`
7. `70-report-compiler`

## Как использовать

Основной вход для пользователя - агент `10-delivery-manager`.

## Режимы запуска

- Если сообщение пользователя начинается с `Задача:`, это команда на запуск полного конвейера разработки.
- Формат: `Задача: <бизнесовая постановка>`.
- Для `Задача:` обязательно создается папка отчета в `.ai/reports/tasks/<date>_<slug>/` и дальше выполняются все этапы pipeline.
- Если сообщение не начинается с `Задача:`, считай его прямой командой пользователя.
- Прямая команда не обязана запускать агентный конвейер и может исполняться как обычная локальная правка, ответ на вопрос или разовое изменение.

Пользователь формулирует только бизнесовую задачу, например:

`Сделай ручку список пользователей с доступом по http-basic для администраторов в system папку`

Пример запуска конвейера:

`Задача: Сделай ручку список пользователей с доступом по http-basic для администраторов в system папку`

Дальше `10-delivery-manager` сам обязан:

1. Прочитать `.ai/rules/core.mdc`.
2. Прочитать `.ai/architecture/instruction.md`.
3. Прочитать нужные layer-файлы из `.ai/architecture/`.
4. Сформировать архитектурный контекст.
5. Разбить задачу на технические шаги.
6. Внедрить код.
7. Написать тесты.
8. Провести ревью.
9. Собрать отчет в `.ai/reports/tasks/<date>_<slug>/`.
