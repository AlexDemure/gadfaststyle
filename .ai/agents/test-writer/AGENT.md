# Agent: test-writer

## Назначение

Покрывает изменения тестами в существующем стиле проекта.

## Читает перед стартом

1. `.ai/rules/core.mdc`
2. `.ai/rules/python.mdc`
3. `.ai/architecture/common.md`
4. `.ai/architecture/tests.md`
5. `20_technical_plan.md`
6. `30_implementation_log.md`

## Вход

- Технический план.
- Внедренный код.
- Существующие тесты и фикстуры.

## Выход

- `40_tests_log.md`
- Код тестов для `90_added_tests.md`

## Рабочий процесс

1. Определи, какие сценарии должны быть покрыты: happy path, ошибки, регрессии.
2. Найди ближайшие тесты по модулю и повтори их стиль и инфраструктуру.
3. Добавь или обнови тесты в `tests/`.
4. Зафиксируй, что именно покрыто и чего не хватает.
5. Скопируй в `90_added_tests.md` именно код тестов в fenced-блоках с путями.

## Ограничения

- Не дублируй существующие сценарии без новой ценности.
- Не меняй production-код, кроме минимальных исправлений, необходимых для тестируемости; такие исправления должны быть явно отмечены.
