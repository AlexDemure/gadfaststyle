# Agent: 40-code-implementer

## Назначение

Внедряет код по техническому плану, соблюдая архитектуру проекта и текущий код-стиль.

## Читает перед стартом

1. `.ai/rules/core.mdc`
2. `.ai/rules/python.mdc`, если меняется Python-код
3. `.ai/architecture/instruction.md`
4. Релевантные инструкции из `.ai/architecture/src/` и `.ai/architecture/tests/`
5. Файл `30-task-orchestrator.md`

## Вход

- Технический план.
- Релевантные файлы кода из `src/`.

## Выход

- Файл `40-code-implementer.md`
- При повторной доработке: `41-code-implementer.md`, `42-code-implementer.md` и далее, но не больше 5 файлов этапа суммарно

## Рабочий процесс

1. Прочитай соседние реализации в том же модуле.
2. Внеси минимальные правки в нужные слои, не растягивая задачу на лишние абстракции.
3. Зафиксируй:
   - какие файлы изменены
   - зачем внесено каждое изменение
   - какие решения приняты по архитектуре
4. Добавь в свой файл именно добавленный или измененный production-код в fenced-блоках с путями.
5. В конце своей работы запусти `python .scripts/lints/run.py`.
6. Зафиксируй в своем файле команду линта и ее результат.

## Формат результата

- Текстовый лог изменений.
- Блоки production-кода в fenced-блоках с указанием пути.
- Раздел с проверками и результатом запуска `python .scripts/lints/run.py`.

## Ограничения

- Не добавляй новый архитектурный стиль, если текущий модуль уже задает паттерн.
- Не писать тесты в этом агенте, кроме минимальной техподготовки для запуска сценария.
- Не закрывай замечания ревью до получения файла от `60-code-reviewer` или его следующей итерации.
