# Формат отчетов

Папка отчета создается только для запросов, запущенных через префикс `Задача:`.

Каждая задача хранится в отдельной папке:

`.ai/reports/tasks/<YYYY-MM-DD>_<slug>/`

Минимальный набор файлов внутри папки:

- `10-delivery-manager.md` - исходная постановка и запуск конвейера
- `20-project-architect.md` - архитектурный контекст
- `30-task-orchestrator.md` - технический план
- `40-code-implementer.md` - внедрение кода
- `50-test-writer.md` - покрытие тестами
- `60-code-reviewer.md` - замечания ревью
- `70-report-compiler.md` - итоговый собранный отчет
- Дополнительные итерационные файлы при необходимости:
  - `41-code-implementer.md`, `42-code-implementer.md`
  - `61-code-reviewer.md`, `62-code-reviewer.md`
  - аналогично для других цикличных этапов

## Правила

- Одна задача - одна отчетная папка.
- Один агент - один файл текущей итерации.
- Итерации одного этапа нумеруются внутри своего десятка.
- Для одного этапа допускается не больше 5 файлов суммарно, включая первую итерацию.
- Каждый агент должен прикладывать в свой файл реальный результат своей работы, а не только краткое summary.
- `20-project-architect.md` должен содержать todo-list в виде таблицы `Status | Executor | Description`.
- Файлы внедрения кода и тестов должны содержать как текстовый лог, так и code-блоки с указанием пути к файлу, если агент добавлял код.
- Если агент запускал проверки, команды и результат должны быть зафиксированы в его файле.
- Финальный файл `70-report-compiler.md` не сжимает историю до summary, а собирает полный отчет по предыдущим файлам.
- Если задача не дошла до тестов или ревью, соответствующий файл все равно создается с явной пометкой причины.
