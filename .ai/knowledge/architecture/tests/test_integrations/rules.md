## Правила

- В первую очередь покрывай HTTP-сценарии интеграционными тестами.
- Тесты размещай в зеркальной структуре относительно production-кода.
- Для HTTP используй `httpx.AsyncClient`.
- Для доступа к БД используй `AsyncSession` и существующие фикстуры.
- Если появляется новый домен, создавай новую папку `test_<domain>`.
- Если после удаления функциональности тестовый пакет пустеет, удаляй пустую папку.
- Тест оформляй в стиле `given / when / then`.

## Запрещено

- Не подменяй интеграционный HTTP-сценарий unit-тестом endpoint-а.
- Не дублируй существующий happy path без новой проверки.
