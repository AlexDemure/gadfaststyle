## Правила

- Используй `@sessionmaker.read`.
- Query-usecase не меняет состояние сущности и не содержит write-операций.
- Query-usecase держит только чтение и минимальную orchestration-логику вокруг него.
- Для чтения одной сущности используй `get`.
- Для чтения коллекции используй `search`.

## Допустимо

- Нормализация входных параметров перед чтением.
- Пост-проверки состояния после чтения, если они бизнесовые.
- Пост-обработка результата перед `return`, если она не превращает query в mutation.
