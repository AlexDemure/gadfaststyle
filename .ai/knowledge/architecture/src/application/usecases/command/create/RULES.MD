## Правила

- Для конфликта существования используй явный паттерн доменной ошибки `{Model}AlreadyExists`.
- Создавай доменную модель через `Model.init(...)` или другой уже принятый domain factory-паттерн.
- Если модель использует `__encrypted__`, usecase должен перед созданием передать в `Model.init(...)` уже подготовленные зашифрованные значения или вызвать предусмотренный моделью helper для шифрования.
- Если `create` возвращает доменную модель, а ее поля хранятся в зашифрованном виде, перед возвратом usecase должен вернуть модель в дешифрованной форме.

## Допустимо

- Проверка существования через repository `exists(...)`.
- Шифрование, нормализация и вычисление производных полей перед созданием модели.
- Дешифрование доменной модели перед возвратом из usecase, если контракт сценария возвращает именно модель.
- Возврат токенов, DTO или модели, если это уже соответствует форме домена.

## Частные случаи

- Если доменная модель уже умеет `encrypt(...)`, `decrypt(...)`, `normalize(...)` или похожую подготовку, используй этот слой, а не дублируй преобразование вручную в нескольких местах.
