## Правила

- Используй `@sessionmaker.write`.
- Для предварительных проверок добавляй отдельный `validate(...)`.
- Если внутри `validate(...)` есть awaitable-операции, делай его `async def validate(...)`.
- Если внутри `validate(...)` нет awaitable-операций, делай его обычным `def validate(...)`.
- `validate(...)` ничего не возвращает и принимает только аргументы, необходимые для проверки.
- Проверки конфликтов, существования и допустимости переходов состояния выполняй до mutation-операции.
- Mutation-usecase держит orchestration: проверки, security, нормализация входа, вызовы repository.

## Допустимо

- Шифрование, нормализация и вычисление производных полей перед записью.
- Пост-обработка доменной модели перед возвратом, если это требуется контрактом сценария.
