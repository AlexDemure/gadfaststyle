## Правила

- Usecase работает с repository adapter, а не с CRUD напрямую.
- Для каждого домена создавай отдельный repository adapter file.
- Adapter связывает CRUD, table, domain model и доменную ошибку.
- Для бизнес-ручек `search` добавляй в adapter отдельный метод `search(...)`, который делегирует поиск в CRUD.
- Файл adapter определяется доменом.
- Repository adapters одного домена храни в одном файле домена.
- Если дополнительная persistence-сущность относится к тому же домену, ее adapter-логика остается в том же доменном файле.
- Если сущность образует отдельный домен, выноси ее adapter в отдельный файл, даже если она связана с другим доменом.
- Публичные adapters одного подслоя экспортируй через `__init__.py` с явными импортами и `__all__`.
- Если пакет собирает не отдельные классы, а целый подслой, экспортируй сам подслой, как `adapters -> repositories`.
- Внутри adapter-модулей импортируй слои целиком, когда пакет уже собирает их через `__init__.py`: `models`, `tables`, `crud`, `exceptions`.

## Запрещено

- Не переноси в adapter бизнес-логику usecase.
