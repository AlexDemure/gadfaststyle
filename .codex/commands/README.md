## Назначение

`.codex/commands` хранит команды Codex и описание логики их выполнения.

## Состав

- [codegen/README.md](./codegen/README.md) - полный цикл codegen: от бизнес-задачи до spec prompt-ов, генерации кода, тестов и финального отчета.

## Правила

- Команды описывают orchestration-логику, а не проектные знания.
- Проектные правила и code style читаются из `.instructions/`.
- Spec prompt-ы читаются и записываются в `.specs/`.
- Артефакты выполнения сохраняются в `.codex/tasks/`.
