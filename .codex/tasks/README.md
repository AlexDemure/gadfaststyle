## Назначение

`.codex/tasks` хранит артефакты работы ИИ-агента по каждой задаче.

## Структура

- Для каждой задачи создается папка `.codex/tasks/<YYYY-MM-DD>_<slug>/`.
- Внутри папки каждый этап пишет отдельный markdown-файл.
- Базовые имена этапов: `10-system-analyst.md`, `20-backend.md`, `30-tester.md`, `40-team-lead.md`.
- При повторных циклах доработки используй следующий свободный номер внутри десятка: `21-backend.md`, `31-tester.md`, `41-team-lead.md` и так далее.

## Правила

- Один запуск задачи должен писать артефакты только в одну папку задачи.
- Артефакт `system-analyst` фиксирует постановку задачи и spec prompt-ы.
- Артефакт `team-lead` фиксирует приемку, замечания или финальный отчет.
- Если этап не выполнен, вместо пропуска создавай файл с блокером или явной причиной.
- Для циклических этапов не создавай больше 5 файлов на один тип этапа, включая первую итерацию.
- Отчетные файлы должны быть пригодны и для чтения человеком, и для последующей автоматической обработки.
