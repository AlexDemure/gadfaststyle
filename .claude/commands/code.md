## Процесс

Выполни полный цикл разработки по бизнес-задаче, следуя этому процессу:

1. Все участники читают релевантные разделы из `.instructions/` и соблюдают правила проекта.
2. Действуй как **системный аналитик**: проанализируй задачу и создай, обнови, удали или перемести `spec.md` в `.specs/`.
3. Проставь даты и время в измененных `spec.md`.
4. Действуй как **backend-разработчик**: реализуй код в `src/` на основе соответствующих `spec.md`.
5. Действуй как **тестировщик**: реализуй тесты в `tests/` на основе соответствующих `spec.md`.
6. При изменениях в коде и тестах обновляй даты и время в соответствующих `spec.md`.
7. Действуй как **тимлид**: проверь результат работы в `src/` и `tests/` на основе `spec.md`, проведи code review и дай обратную связь.
8. Если нужны правки, повторно внеси изменения и продолжи дописывать runtime-файл.
9. Если правки не нужны, подведи итог работы.
10. Сохрани один runtime-файл в `.claude/runtime/code/<YYYY-MM-DD_HH-MM-SS>_<slug>.md`.

## Требования

- Один путь проекта соответствует одному `spec.md`.
- Backend работает только в `src/`.
- Тестировщик работает только в `tests/`.
- Тимлид проверяет результат по `spec.md`, проектным инструкциям и фактическим изменениям в `src/` и `tests/`.
- Если по задаче требуется несколько итераций, весь процесс дописывается в один и тот же runtime-файл.
- Во всех измененных `spec.md` должны быть проставлены актуальные дата и время с секундами.

## Формат runtime-файла

Runtime-файл должен начинаться со строки `Разработка: YYYY-MM-DD HH:MM:SS`.

Runtime-файл ведется последовательно и дополняется по ходу работы. Каждый этап оформляй отдельным заголовком с номером шага. Формат заголовка: `## №<n> <роль> - <этап>`.

Для каждого `spec.md` используй заголовок в формате `edited:<path>`, `created:<path>`, `moved:<old> -> <new>`, `deleted:<path>`. Для `edited`, `created` и `moved` показывай полную актуальную версию `spec.md`. Для `deleted` достаточно только заголовка.
