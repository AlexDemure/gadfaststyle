## Процесс

Выполни полный цикл разработки по бизнес-задаче, следуя этому процессу:

1. Все участники читают `.instructions/src.md`, `.instructions/tests.md` и `.instructions/spec.md` и соблюдают правила проекта.
2. Действуй как **системный аналитик**: проанализируй задачу и создай или обнови feature-спеки в `.specs/<domain>/` по формату из `.instructions/spec.md`.
3. Действуй как **backend-разработчик**: реализуй задачи из раздела `## Backend` соответствующего `spec.md` в `src/`.
4. Действуй как **тестировщик**: реализуй задачи из раздела `## Testing` соответствующего `spec.md` в `tests/`.
5. Действуй как **тимлид**: проверь результат работы в `src/` и `tests/` против `spec.md`, проведи code review и дай обратную связь.
6. Если нужны правки, внеси изменения и продолжи дописывать runtime-файл.
7. Если правки не нужны, подведи итог работы.
8. Сохрани один runtime-файл в `.claude/runtime/code/<YYYY-MM-DD_HH-MM-SS>_<slug>.md`.

## Требования

- Backend работает только в `src/`.
- Тестировщик работает только в `tests/`.
- Тимлид проверяет результат по `spec.md`, `.instructions/src.md`, `.instructions/tests.md` и фактическим изменениям.
- Если по задаче требуется несколько итераций, весь процесс дописывается в один и тот же runtime-файл.

## Формат runtime-файла

Runtime-файл должен начинаться со строки `Разработка: YYYY-MM-DD HH:MM:SS`.

Runtime-файл ведется последовательно и дополняется по ходу работы. Каждый этап оформляй отдельным заголовком с номером шага. Формат заголовка: `## №<n> <роль> - <этап>`.

Для каждого `spec.md` используй заголовок в формате `edited:<path>`, `created:<path>`, `deleted:<path>`. Для `edited` и `created` показывай полную актуальную версию `spec.md`. Для `deleted` достаточно только заголовка.
